File Comments

You can add comments to any file without changing the original file. Comments are automatically displayed when you preview the file. Comments are hosted by the operating system and are invisible to other software. Moreover, since the comment process does not modify the file in any way, even Microsoft Explorer cannot view the comments. Editing and renaming the file will not affect the file comments.

In actual testing, the operating system's hosting of comment content is not perfect. If the file is moved, copied, or pasted on the local computer, the comment content will not be lost. However, if the file is copied or moved to another computer, the file comment content may be lost. Whether the file comment is lost during the file operation depends on the file operation method. What is certain is that if the carrier of the file operation is the network, the file comment content will definitely be lost. If the carrier of the file operation is a USB flash drive or a mobile hard drive, the file comment content will not be lost. [ MP4 ]

Text file previewer

Automatic recognition of text files

The identification of text files is not determined by the file extension, but by the software automatically analyzing the file content and intelligently identifying the text file type. Therefore, real files with TXT extensions, as well as real C, CPP, CS, CSS, JS and other files will be automatically identified as text files. Even if the extension of an MP4 file is changed to TXT, it will not be identified as a text file.

Manual collaborative preview

Manual collaborative preview

By default, previewers preview files independently. The collaborative preview feature can coordinate the previewers of multiple file explorers in a file explorer group to work together, allowing users to select files in one previewer and preview them in multiple previewers at the same time. Collaborative preview includes manual collaborative preview and automatic collaborative preview. Mirror video and mirror image can bring you a different user experience.

 

Set manual collaborative preview mode

Check the Collaborative Preview button to enter manual collaborative preview mode. If the user selects a file, all other previewers in the group will preview the file.

This feature is valid for 2-resource manager groups, 3-resource manager groups, and 4-resource manager groups. The middle previewer of a 3-previewer group never participates in collaborative preview.

If the user selects more than one file, the other previews in the group will preview the first selected file first. Then, the second previewer will preview the second selected file, the third previewer will preview the third selected file (if there is a third selected file), and so on.

 

Image automatic mirroring preview mode

Check the collaborative preview button and then check the mirror button. If you select a picture, you will enter the automatic mirror preview mode. The automatic mirror preview mode of the picture is only valid for pictures, and only one picture needs to be selected. In the automatic mirror preview mode of the picture, when you preview a picture, the mirror image of the picture will be automatically previewed in other previewers in the group. This function is valid for 2-explorer group, 3-explorer group and 4-explorer group.

 

Video automatic mirroring preview mode

Check the Co-Preview button and then check the Mirror button. If you select a video, you will enter the video automatic mirror preview mode. The video automatic mirror preview mode is only valid for videos, and only one video needs to be selected. In the video automatic mirror preview mode, when previewing a video, the mirror video of the video will be automatically previewed in other previewers in the group. This function is valid for 2-explorer group, 3-explorer group and 4-explorer group. In the video automatic mirror preview mode, 2-explorer group and 3-explorer group need to prepare horizontal mirror files. 4-explorer group needs to prepare all 3 mirror files.

 

Mirror video files

The automatic video mirroring preview mode requires that the mirrored video must be prepared in advance in the same directory as the main video. And the mirrored video name must meet the following requirements.

Naming rules for mirrored videos:

Assume the original video file name is: original video.mp4,

The horizontal image is: original file name_HF.mp4,

Vertical mirror image: original file name_VF.mp4,

The vertical and horizontal mirror images are: original file name_VH.mp4.
